### 实验 5：静态路由配置执行步骤


### 第一步：硬件与连线（严格按此端口选择）

1. **放置设备**：3台 `1841` 路由器（命名为R0, R1, R2），2台 `PC`（PC1, PC3）。
    
2. **安装模块**：点击所有路由器 -> 关闭电源 -> 将 `WIC-2T` 拖入**右侧插槽（Slot 0）** -> 打开电源。（这样串口编号会统一为 `Serial0/0/0` 和 `Serial0/0/1`）。
    
3. **严格连线**：
    
    - PC1 `FastEthernet0` <--交叉线--> R0 `FastEthernet0/0`
        
    - PC3 `FastEthernet0` <--交叉线--> R2 `FastEthernet0/0`
        
    - R0 `Serial0/0/0` <--串口线(DCE红表)--> R1 `Serial0/0/0` （先点R0）
        
    - R2 `Serial0/0/0` <--串口线(DCE红表)--> R1 `Serial0/0/1` （先点R2）
        

---

### 第二步：PC 端 IP 配置

- **PC1**：IP `192.168.1.100`，掩码 `255.255.255.0`，网关 `192.168.1.1`
    
- **PC3**：IP `192.168.3.100`，掩码 `255.255.255.0`，网关 `192.168.3.3`
    

---

### 第三步：路由器全量配置（直接复制粘贴）

**【R0 路由器配置】**

Plaintext

```
enable
configure terminal
! 1. 配置接口
interface fa0/0
ip address 192.168.1.1 255.255.255.0
no shutdown
exit
interface serial0/0/0
ip address 10.1.1.1 255.255.255.252
clock rate 64000
no shutdown
exit
! 2. 配置去程静态路由
ip route 192.168.3.0 255.255.255.0 10.1.1.2
ip route 10.1.2.0 255.255.255.252 10.1.1.2
end
```

**【R1 路由器配置】**

Plaintext

```
enable
configure terminal
! 1. 配置接口
interface serial0/0/0
ip address 10.1.1.2 255.255.255.252
no shutdown
exit
interface serial0/0/1
ip address 10.1.2.1 255.255.255.252
no shutdown
exit
! 2. 配置双向静态路由
ip route 192.168.1.0 255.255.255.0 10.1.1.1
ip route 192.168.3.0 255.255.255.0 10.1.2.2
end
```

**【R2 路由器配置】**

Plaintext

```
enable
configure terminal
! 1. 配置接口
interface fa0/0
ip address 192.168.3.3 255.255.255.0
no shutdown
exit
interface serial0/0/0
ip address 10.1.2.2 255.255.255.252
clock rate 64000
no shutdown
exit
! 2. 配置回程静态路由
ip route 192.168.1.0 255.255.255.0 10.1.2.1
ip route 10.1.1.0 255.255.255.252 10.1.2.1
end
```

---

### 第四步：终极测试

1. 观察拓扑图，**所有连线必须全为绿点**。如果有红点，检查线是否插错端口。
    
2. 打开 **PC1** 的 Command Prompt。
    
3. 输入 `ping 192.168.3.100`。
    
4. 第一个包可能会 `Request timed out`，后面三个必然是 `Reply from 192.168.3.100...`。

### 实验 6：RIP 动态路由协议

**第一步：清除原本的静态路由** 进入各自的全局配置模式（ `configure terminal` ），删除实验 5 的静态路由：

- **R0输入**: `no ip route 192.168.3.0 255.255.255.0 10.1.1.2` `no ip route 10.1.2.0 255.255.255.252 10.1.1.2`
- **R1输入**: `no ip route 192.168.1.0 255.255.255.0 10.1.1.1` `no ip route 192.168.3.0 255.255.255.0 10.1.2.2`
- **R2输入**: `no ip route 192.168.1.0 255.255.255.0 10.1.2.1` `no ip route 10.1.1.0 255.255.255.252 10.1.2.1`

**第二步：配置 RIPv2 协议**

**【R0 输入】**

```markdown
configure terminal
router rip
version 2
no auto-summary
network 192.168.1.0
network 10.0.0.0
end
```

**【R1 输入】**

```markdown
configure terminal
router rip
version 2
no auto-summary
network 10.0.0.0
end
```

**【R2 输入】**

```markdown
configure terminal
router rip
version 2
no auto-summary
network 192.168.3.0
network 10.0.0.0
end
```

**第三步：验证测试**

1. 在 R1 输入 `show ip route` ，确认学习到了前面带 **`R`** 的路由条目。
2. 在 PC1 的命令行输入 `ping 192.168.3.100` ，等待几秒后应提示连通（Reply）。

### 实验 7：单区域 OSPF 路由协议

继续使用当前拓扑，先清理 RIP，再配置 OSPF。

**第一步：清除 RIP 协议** 分别在 **R0、R1、R2** 的全局配置模式（ `configure terminal` ）下输入： `no router rip`

**第二步：配置 OSPF 协议**

**【R0 输入】**

```markdown
configure terminal
router ospf 1
network 192.168.1.0 0.0.0.255 area 0
network 10.1.1.0 0.0.0.3 area 0
end
```

**【R1 输入】**

```markdown
configure terminal
router ospf 1
network 10.1.1.0 0.0.0.3 area 0
network 10.1.2.0 0.0.0.3 area 0
end
```

**【R2 输入】**

```markdown
configure terminal
router ospf 1
network 192.168.3.0 0.0.0.255 area 0
network 10.1.2.0 0.0.0.3 area 0
end
```

**第三步：验证测试**

1. 命令行会弹出 `...from LOADING to FULL, Loading Done` 的邻居建立成功提示。
2. 在路由器输入 `show ip route` ，确认学习到了前面带 **`O`** 的路由条目。
3. 在 PC1 执行 `ping 192.168.3.100` 确认全网连通。

### 实验 8：DHCP 服务配置

这是一个全新的独立实验，请 **File -> New** 新建一个空白文件。

**第一步：硬件连线**

1. 放入 1 台路由器（如 1841）、1 台交换机（如 2960）、1 台 Server-PT（服务器）和 2 台 PC。
2. 用直通线（Straight-Through）将 PC 和 Server 都连到交换机上。
3. 用 **直通线** 将交换机连到路由器的 `FastEthernet0/0` 接口。

**第二步：配置 Web 服务器 (Server0)**

1. 点击 Server0 -> `Desktop` -> `IP Configuration` ：
	- **IP**: `192.168.1.10`
		- **掩码**: `255.255.255.0`
		- **网关**: `192.168.1.1`
2. 点击 `Services` -> `HTTP` ，确保 HTTP 服务状态为 **ON** 。

**第三步：配置路由器 (作为 DHCP 服务端)**

**【Router 输入】**

```markdown
enable
configure terminal
! 1. 配置网关接口
interface fa0/0
ip address 192.168.1.1 255.255.255.0
no shutdown
exit

! 2. 排除不分配的地址（网关和服务器地址）
ip dhcp excluded-address 192.168.1.1 192.168.1.20

! 3. 创建 DHCP 地址池
ip dhcp pool LAN
network 192.168.1.0 255.255.255.0
default-router 192.168.1.1
dns-server 8.8.8.8
end
```

**第四步：验证测试**

1. 点击 PC -> `Desktop` -> `IP Configuration` ，选择 **DHCP** 选项。
2. 看到提示 `DHCP request successful` ，且分配到的 IP 是 `192.168.1.21` （因为前 20 个被排除了）。
3. 打开 PC 的 `Web Browser` ，输入服务器 IP： `192.168.1.10` 并回车，成功看到 Packet Tracer 的默认网页即代表实验圆满完成。
