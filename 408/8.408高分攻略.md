# 408高分攻略

## 一、认清408：一场高性价比的信息战

408总分150分，考试180分钟。四门课分值分布：

| 科目      | 分值  | 题型分布                 |
| ------- | --- | -------------------- |
| 数据结构    | 45分 | 选择题 + 综合应用题（算法设计）    |
| 计算机组成原理 | 45分 | 选择题 + 综合应用题（硬件分析/设计） |
| 操作系统    | 35分 | 选择题 + 综合应用题（概念+计算）   |
| 计算机网络   | 25分 | 选择题为主+一个综合体          |

**核心认知**：408不是考察你有多聪明，而是考察你对**高频考点**的熟练度。根据18年真题统计，35个高频考点（占知识点总数36.1%）贡献了超过60%的分数。抓住这些考点，130+触手可及。

## 三、分科目高分策略

### 3.1 数据结构（45分）

**得分分布**：选择题约22分 + 综合题约23分（含算法设计题）

**核心打法**：
1. **算法题是拉开差距的关键**（10-15分综合题）。练习手写伪代码，重点掌握：链表操作（插入、删除、反转）、二叉树遍历（递归与非递归）、图的DFS/BFS、查找与排序算法
2. **时间复杂度分析是送分题也是易错题**。对每种算法的时间/空间复杂度形成条件反射
3. **树和图是综合题主战场**。哈夫曼树构造、平衡二叉树调整、最短路径（Dijkstra/Floyd）、最小生成树（Prim/Kruskal）、拓扑排序、关键路径——这些过程要能熟练模拟
4. **B树/B+树**的操作细节经常作为选择题区分度来源，集中记忆插入/删除的分裂合并过程

**容易翻车的点**：
- 栈的输出序列合法性判定（模拟入栈/出栈过程）
- 折半查找判定树的形态（奇数/偶数结点）
- 散列表的冲突处理（开放定址时的探测次数，ASL成功/不成功计算）
- 算法题审题——看清要求"写出算法思想"还是"给出具体代码"

### 3.2 计算机组成原理（45分）

**得分分布**：选择题约22分 + 综合题约23分

**计组是408最难拿分的科目，也是拉开分差的关键。**

**核心打法**：
1. **Cache-主存-虚拟存储器**是综合题的"铁三角"，三者联合考察已成标准模式。必须熟练：
   - 地址分段（标记/组号/块内偏移）
   - TLB、Cache、Page Table同时命中/缺失的各类组合
   - Cache映射方式与替换算法
2. **数据通路**（2026年高概率考点）——单周期CPU的数据通路图要能自己画出并标注控制信号；重点掌握R型/I型/Load/Store/分支指令在数据通路中的执行过程
3. **指令格式与寻址方式**——操作码扩展技术、地址码计算
4. **定点/浮点运算**——补码加减、溢出判断、IEEE 754浮点数表示与加减步骤
5. **机器级代码**——C语言与汇编的对应关系、过程调用的栈帧变化（push ebp/mov ebp esp）、局部变量/参数的地址计算

**容易翻车的点**：
- 大小端存储与数据对齐（结构体地址对齐）
- 补码加减的CF与OF差异：CF判断无符号溢出，OF判断有符号溢出
- 浮点数规格化与对阶过程
- 大题的读题——给出一段C代码+汇编或机器码，分析每条指令的执行

### 3.3 操作系统（35分）

**得分分布**：选择题约18分 + 综合题约17分

**核心打法**：
1. **PV操作**是操作系统综合题的第一选择（15年106分）——必须熟练写出生产者-消费者、读者-写者、哲学家进餐的PV伪代码，注意互斥信号量与同步信号量的顺序
2. **虚拟内存管理**（18年166分，最高分考点）——缺页中断处理流程、页面置换算法（FIFO/LRU/OPT/改进Clock）、Belady异常、工作集模型
3. **文件系统**（17年92分）——索引节点（直接/间接索引）、文件物理结构（连续/链接/索引）、空闲空间管理（位示图/空闲链表）
4. **处理机调度**（18年58分）——各类调度算法的**平均周转时间**和**平均等待时间**计算，这是送分题

**容易翻车的点**：
- 虚拟内存+Cache+TLB三者联合计算访问时间
- PV操作中的信号量初始值设定
- 死锁的银行家算法计算（Available/Need/Allocation矩阵）
- 磁盘调度算法的寻道距离计算（注意方向与时序）

### 3.4 计算机网络（25分）

**得分分布**：选择题约16分 +  综合题9分

**核心打法**：
1. **IPv4子网划分与CIDR**（15年114分）——给定IP地址块，划分子网，写出各子网的网络地址、子网掩码、可用IP范围。这是必拿的基础分
2. **TCP协议**（16年72分）——序号/确认号计算（三次握手和正常数据传输）、拥塞控制窗口变化图（慢开始→拥塞避免→快重传→快恢复）、TCP与UDP的头部字段比较
3. **局域网**（13年42分）——CSMA/CD最小帧长计算、交换机自学习过程、VLAN概念
4. **路由协议**（9年41分，含RIP/OSPF/BGP）——三种协议的核心区别（距离向量vs链路状态、AS内部vs AS之间、UDP vs IP承载）
5. **数据链路层可靠传输**（11年29分）——三种协议的发送窗口/接收窗口大小、序号空间
6. **OSI与TCP/IP模型**——2026大纲特别标注"26改动"，五位一体（各层PDU名称、核心协议、核心功能）要达到条件反射

**容易翻车的点**：
- 计算TCP序号时漏掉三次握手的SYN占用一个序号
- CSMA/CD冲突窗口 = 2τ（往返传播时延）
- RIP最大跳数16（不可达），对链路变化反应慢（坏消息传得慢）
- DNS递归查询与迭代查询的区别

## 四、408综合题突破——跨科目联考已成趋势

近年408综合题越来越倾向**跨科目联合命题**，以下组合必须会：

### 经典联考模式

1. **C程序 → 机器码 → 数据通路执行 → 流水线冒险分析**
   - 给出C代码片段和对应的MIPS/x86汇编
   - 分析每条指令的数据通路和控制信号
   - 分析流水线中的数据冒险（转发/阻塞）和控制冒险（分支预测）
   - 涉及：计组+部分OS内存管理知识

2. **虚拟地址 → TLB查找 → Cache查找 → 主存访问 → 缺页处理**
   - 给一个虚拟地址，逐级分析TLB命中/缺失、Cache命中/缺失、Page Fault
   - 计算总访问时间（命中/缺失各场景的加权平均）
   - 涉及：计组（Cache+TLB）+ OS（虚拟内存）

3. **进程同步 → 共享内存 → Cache一致性**
   - PV操作控制多进程/线程访问共享数据
   - 涉及：OS（同步互斥）+ 计组（Cache一致性基础）

## 五、复习时间规划与节奏

### 建议总时长：4-6个月（每天3-4小时）

| 阶段 | 时间 | 目标 | 方法 |
|------|------|------|------|
| **基础夯实** | 第1个月 | 扫完四门课教材 | 按科目顺序过一遍（DS→计组→OS→网络），每章做课后选择题 |
| **专题深化** | 第2个月 | 攻克35个高频考点 | 每个考点找3-5道真题练手，建立错题本 |
| **真题精练** | 第3-4个月 | 18年真题过2遍 | 第一遍按年份做，第二遍按知识点横向分类刷 |
| **冲刺模拟** | 第5-6个月 | 查漏补缺+综合训练 | 限时模拟，控制选择题40分钟，综合题50分钟 |

### 每日时间分配建议

- 60%时间给**计组+DS**（各30%）——这两门是拉开差距的科目
- 30%时间给**OS**——分值高且稳定，性价比好
- 10%时间给**网络**——分值少且多为记忆性内容，碎片时间搞定

## 六、考场策略——180分钟如何分配到150分

### 6.1 时间分配

| 题型        | 建议时间    | 说明                              |
| --------- | ------- | ------------------------------- |
| 选择题（40题）  | 40-50分钟 | 每题≤1分钟，不会的标记跳过后回头               |
| 综合题（7道左右） | 70-80分钟 | 每题10-12分钟，优先做自己擅长的              |
| 检查        | 10-20分钟 | 重点检查易算错的计算题（Cache地址、IP子网、调度时间等） |

### 6.2 做题顺序

**推荐顺序**：选择题按顺序做 → 综合题按**擅长程度**排序做（优先做OS、网络的大题，再做DS算法设计，最后攻计组数据通路）

理由：OS和网络综合题相对固定模式，容易拿分；DS算法题需要思考时间但一旦想通很快写成；计组数据通路是大题中最耗时也最容易出错的。

### 6.3 选择题提速技巧

- 对高频考点形成**条件反射**（例如：看到"平均查找长度最小的排序算法"立刻想到快速排序）
- **排除法**在概念题上的妙用——两个矛盾选项必有其一是答案
- 遇到需要精确计算的题（Cache地址分段、浮点数加减），先在草稿纸上列好公式再代值

## 七、避坑清单

1. **别迷恋视频课**——看课不刷题=没复习。每看1小时课至少配合1小时刷题
2. **别在低频考点上花时间**——27个低频考点加在一起也就20-30分，花同样时间在高频考点上收益翻倍
3. **别忽视真题**——18年真题至少完整做2遍。408命题有明显的延续性和套路
4. **别忽视计算**——计组和OS大题几乎都要算，计算能力不过关直接丢链
5. **别放弃计组**——计组是四科中最难的，但45分的权重和DS相同。放弃计组等于放弃高分
6. **别信"网络不重要"**——25分看似少，但网络是408中最容易拿满分的科目，25分全拿和只拿15分差别巨大
7. **算法题别追求最优解**——408综合题的时间限制下，写出**正确的暴力解**优于**有bug的最优解**。先保证正确性再考虑优化
8. **别忽视大纲变化**——2026大纲标注"26改动"的OSI/TCP/IP参考模型是重点关注方向

## 八、总结——408高分的三个层次
