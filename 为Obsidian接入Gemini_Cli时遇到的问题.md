# 终端接入 Gemini CLI 全记录：代理排坑与终极配置指南

## 🌟 核心目标
在本地终端（PowerShell）中接入 Google 官方的 `@google/gemini-cli`，彻底告别浏览器切换，实现沉浸式的代码辅助与本地笔记分析功能。

## 🛠️ 环境准备
- 已安装 Node.js 与 npm。
- 全局安装 CLI 工具：`npm install -g @google/gemini-cli`。
- 本地拥有科学上网环境（记下代理软件的本地端口，例如：`6984`）。

---

## 💥 踩坑记录与原理解析

### 坑 1：`gemini login` 网页授权的“死锁”
- **现象**：在终端输入 `gemini login` 呼出浏览器，网页端点击同意授权后，终端依然卡在 `Waiting for authentication...`，直至超时失败。
- **底层原因**：**本地代理环回劫持**。浏览器拿到授权码后，试图发回给本地终端监听的端口（如 `http://127.0.0.1:59452`），但本地代理软件强行将发往本地的流量也代理到了海外，导致终端永远收不到“回执”。
- **破局方案**：直接放弃网页授权，改用 **API Key 环境变量**认证。

### 坑 2：获取 API Key 时的“双重卡点”
前往 [Google AI Studio](https://aistudio.google.com/) 获取 API Key 时，通常会遇到两道坎：
1. **地区限制拦截**：提示 `Not available in your country/region`。
   - **解决**：必须将代理节点切换为**美国 (US)、日本 (JP) 或新加坡 (SG)** 等支持地区，并开启浏览器的**无痕模式**重新访问。
2. **“Create key” 按钮置灰**：提示 `No Cloud Projects Available`。
   - **解决**：这是因为 Google 账号下没有任何 Cloud 项目。需新开网页前往 [Google Cloud 控制台](https://console.cloud.google.com/)，同意服务条款并随意**新建一个项目**。返回 AI Studio 刷新页面，即可选中项目并顺利生成一长串 API Key。

### 坑 3：终端里 AI 的“局部失忆”与代理缺失
- **现象**：顺利配好 API Key 后，在 `gemini>` 聊天界面提问，AI 却卡在 `Thinking...` 长达数分钟不回复。
- **底层原因**：PowerShell 终端默认**不会继承**系统的代理设置。AI 正在试图用国内直连网络硬连 Google 服务器，导致被墙卡死。
- **致命盲区**：如果在 `gemini>` 的聊天提示符下输入配置代理的命令，AI 会以为你在和它“聊天探讨代码”，并不会真正执行系统配置！

## 🚀 终极通关指南（一劳永逸配置法）

为了避免每次打开终端都要配置一遍，强烈推荐将密钥和代理写入 Windows 的系统环境变量：

1. 按 `Win` 键搜索并打开 **“编辑系统环境变量”** -> 点击 **“环境变量”**。
2. 在上半部分的 **“用户变量”** 中，点击“新建”，依次添加以下三个变量：
   - 变量名：`GEMINI_API_KEY`  | 变量值：`你的那一长串API_Key`
   - 变量名：`HTTP_PROXY`    | 变量值：`http://127.0.0.1:6984` *(注：端口号换成你自己的)*
   - 变量名：`HTTPS_PROXY`   | 变量值：`http://127.0.0.1:6984`
3. 一路点击“确定”保存设置。
4. **重新打开 PowerShell**，直接输入 `gemini`，即可实现秒连秒回！

## 💡 进阶玩法：联动本地文件与 Obsidian
配置好 CLI 后，它的真正威力在于直接读取本地文件：

> [!tip] 喂给它长文件
> 在 `gemini>` 聊天界面，不需要复制粘贴长篇代码。直接使用 `@` 符号加上本地文件路径即可：
> `> @D:\jx_repo\某个代码.cpp 这段代码的时间复杂度是多少？帮我优化一下。`
> `> @D:\笔记\数据库原理.md 根据这篇 Obsidian 笔记，帮我出五道复习题。`

> [!info] 换行快捷键
> 在终端内如果想粘贴多行内容，直接按 `Enter` 会发送消息。需要换行请使用 `Shift + Enter` 或 `Alt + Enter`。
