# 408 高分策略

## 1. 回归教材，不要只刷辅导书

王道/天勤总结的是高频考点，但新题往往从教材的边角细节出。四本教材（谢希仁网络、唐朔飞计组、严蔚敏数据结构、汤子瀛操作系统）里的**课后习题、附注、小字部分**都可能是新题来源。尤其是计组和操作系统，很多冷门题直接来自教材原文。

## 2. 建立知识网络而非孤立记忆

比如看到"页表"，脑子要同时串起来：

- 操作系统：页表结构、多级页表、TLB、缺页中断
- 计组：页表基址寄存器、MMU 地址转换、Cache 与 TLB 的关联

**新题往往考的是交叉地带**——两个学科的交界处，任何一本辅导书都不会单独总结。你自己画横向联系图比背十遍都管用。

## 3. 用"为什么"逼自己深入

每个知识点问三层：

- 这个机制解决了什么问题？
- 如果没有它，会怎么出错？
- 它和其他机制是怎么配合的？

例如："中断隐指令"为什么要由硬件完成？——如果让软件做，保护现场时 PC 已经被覆盖了。想通这一层，不管题目怎么变，核心逻辑不会变。

## 4. 真题反复做，但换个角度

同一道真题：

- 第一遍：正常做，看对错
- 第二遍：如果我是出题人，这道题还能怎么变？选项改一个条件，答案会不会变？
- 第三遍：这道题涉及的知识点，在另一门课里怎么考的？

## 5. 针对新题型的专项训练

- 搜集近 2-3 年的真题和模拟题中的**偏题、新题**，分析出题思路变化趋势
- 408 近年趋势是**更重原理推导而非计算**，比如计组的数据通路分析、操作系统的同步语义理解
- 关注 CSAPP（深入理解计算机系统）的风格——408 出题组越来越向这个方向靠拢

---

**一句话总结：用理解深度对抗题目变化，用知识网络覆盖冷门考点。**

# 408 各科重难点及攻克优先级

## 数据结构（45分）

**重难点：**
- **树与图的应用**（每年必考大题）：二叉排序树、平衡二叉树、哈夫曼树、最小生成树、最短路径、拓扑排序、关键路径
- **查找**：B 树 / B+ 树的插入删除、哈希表冲突处理（二次探测法容易算错）
- **排序**：各种排序的趟数判断、堆排序的建堆与调整（手写代码 8-10 分）
- **算法设计题**（最拉分的 10-15 分）：以链表和二叉树为主，近年偶尔出图

**攻克策略：**
- 算法题先练模板三板斧：链表逆置、二叉树遍历（递归→非递归）、图的 DFS/BFS，考场上直接套改
- B 树插入删除画图练到闭眼不错
- 排序复杂度表格烂熟，特别注意 **不稳定排序** 在具体场景下的选择

---

## 计算机组成原理（45分）— 最难的一科

**重难点：**
- **数据通路与控制器**（每年大题，15 分左右）：指令执行过程、数据流向、控制信号分析。这是 408 最难的题型，没有之一
- **Cache 与主存映射**：组相联映射 + LRU 替换 + 写策略，三位一体综合考
- **指令系统**：寻址方式、指令格式设计、CISC vs RISC
- **浮点数 IEEE 754**：规格化、溢出判断、加减运算（对阶步骤易出错）
- **中断与 DMA**：中断隐指令、中断屏蔽字、DMA 与中断的区别和配合

**攻克策略：**
- 数据通路题**必须**拿一张白纸，从取指周期开始，一个节拍一个节拍画出所有微操作和数据流向，练 10 道真题手感就上来
- Cache 大题本质是"翻译地址"——主存地址拆分成标记/组号/块内地址，这一步不错全题就对
- 浮点数运算画双符号位，对阶时往小阶看齐

---

## 操作系统（35分）

**重难点：**
- **PV 操作与同步互斥**（固定大题，8-10 分）：复杂同步问题（读者-写者变体、吸烟者问题、理发师问题）——这是操作系统最能拉开差距的地方
- **内存管理**：分页/分段的地址转换、多级页表计算、页面置换算法（CLOCK 算法细节多）
- **文件系统**：索引节点、多级索引计算最大文件长度、空闲盘块管理（成组链接法）
- **磁盘调度**：SCAN/LOOK/C-SCAN 的方向判断

**攻克策略：**
- PV 题记住一个铁律：**分析几类进程、几类资源、同步还是互斥**，然后画出前驱图，代码自然出来
- 地址转换题把页表画出来，一步步走：逻辑地址 → 页号+偏移 → 查页表 → 物理地址
- 多级索引计算类题画图比公式可靠

---

## 计算机网络（25分）

**重难点：**
- **TCP 拥塞控制与流量控制**（固定大题）：慢启动、拥塞避免、快重传、快恢复的窗口变化曲线
- **IP 协议与路由**：子网划分（CIDR）、RIP/OSPF/BGP 对比、路由表查表匹配（最长前缀匹配）
- **数据链路层**：CSMA/CD 最小帧长计算、后退 N 帧 vs 选择重传的窗口大小和确认机制
- **HTTP/DNS/电子邮件**：应用层协议细节，尤其是 HTTP 各版本对比

**攻克策略：**
- TCP 拥塞窗口变化图**自己从头画到尾**，每个转折点标上导致变化的事件（超时 or 3 个冗余 ACK？）
- 子网划分形成肌肉记忆：/27 就是掩码 224，可用 IP = 32-3=29
- 后退 N 帧 vs 选择重传：一个表格列出窗口大小、确认方式、重传策略，对比例题

## 优先级排序（按投入产出比）

| 优先级 | 科目 | 原因 |
|--------|------|------|
| **T0** | 计组·数据通路 | 大题 15 分，会者不难，不会者全丢 |
| **T0** | 操作系统·PV 操作 | 大题固定 8-10 分，题型有限，练熟拿满分 |
| **T1** | 数据结构·算法题 | 10-15 分，模板化强，性价比高 |
| **T1** | 计算机网络·TCP | 大题固定，考点集中 |
| **T2** | 计组·Cache/浮点 | 变化多但框架固定 |
| **T2** | 操作系统·内存/文件 | 知识点细碎但难度中等 |
| **T3** | 各科细碎选择考点 | 靠教材回归 + 真题覆盖 |

**一句话：计组数据通路和操作系统 PV 是 408 的两大天王山，搞定这两个大题，保底分数就有了；拉开差距靠数据结构算法题和计组冷门细节。**
